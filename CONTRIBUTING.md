# Contributing to Court Control AI

Thank you for your interest in improving the infrastructure for professional sports.

## Development Workflow

1. **Setup**: Run `npm install` and copy `.env.example` to `.env`.
2. **Branching**: Create a feature branch from `main`.
3. **Coding**: Follow the established Prettier and ESLint rules.
4. **Testing**: Run `npm test` to ensure AI flows and UI logic are intact.
5. **Linting**: Run `npm run lint` and `npm run typecheck` before pushing.

## PR Process

- Ensure the CI pipeline passes on your pull request.
- Provide clear descriptions of your changes and any new dependencies.
- Document any updates to the Firestore schema in `docs/backend.json`.
